How it works

Client-Server Setup

You can take any two DuckDB processes on a network and turn them into client and server by using the Quack extension. The Quack protocol supports the full DuckDB feature set – over the wire.

Optimized for Performance

We designed the Quack protocol for performing large analytical queries. We minimized the number of round-trips, and ensured that the protocol can handle small updates and large batches just as well.

Install the Quack extension from the core_nightly repository

  • Command line
  • Python
  • AWS CloudFormation

Step 1

Install DuckDB

{% highlight bash %} curl https://install.duckdb.org | sh {% endhighlight %}

Step 2

Install the Quack extension

{% highlight sql %} INSTALL quack FROM core_nightly; LOAD quack; -- Confirm it loaded SELECT * FROM duckdb_extensions() WHERE extension_name = 'quack'; {% endhighlight %}

Step 3

Start the server

{% highlight bash %} duckdb \ --server quack \ -C /localhost:1234 \ my_database.duckdb # Listening on quack:localhost:1234 {% endhighlight %}
